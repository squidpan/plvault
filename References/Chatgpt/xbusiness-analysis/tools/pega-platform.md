# PEGA Platform

## Summary
- Low-code BPM platform for workflow and case management applications
- Used to model business processes, rules, forms, and routing
- Common in insurance and financial services modernization programs

## Key Points
- Supports workflow orchestration and approval routing
- Uses configuration-driven development rather than traditional full-code development
- Can integrate with enterprise APIs, databases, and external systems
- Requires strong requirements, testing, and release governance

## Interview Insights
- PEGA is not like Spring Boot; it is closer to BPM plus case management plus low-code workflow configuration
- A BA should understand how business rules, workflow states, and user roles affect application behavior
- Production support often involves determining whether an issue is workflow logic, data, integration, or access related

## Real-World Usage
- Underwriting workflow modernization
- Case management applications
- Routing, approvals, and operational workflow automation
