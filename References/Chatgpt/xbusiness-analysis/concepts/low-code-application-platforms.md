# Low-Code Application Platforms

## Summary
- Enable faster delivery of business applications through configuration
- Commonly used for workflow, case management, and business rules
- Require strong governance, testing, and release management

## Key Points
- Use visual workflow models, rules, forms, and integrations
- Reduce custom coding but still require disciplined SDLC practices
- Support rapid workflow modernization and process changes
- Can create operational risk if configuration changes are poorly tested

## Interview Insights
- Low-code does not mean low-discipline
- Business Analysts often play a key role in defining rules, workflow behavior, and acceptance criteria
- QA/UAT is critical because workflow configuration changes can affect many business scenarios

## Real-World Usage
- Building underwriting workflow applications
- Modernizing legacy manual processes
- Supporting PEGA BPM workflow changes
