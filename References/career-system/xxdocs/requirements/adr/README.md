# Architecture Decision Records

Status: Proposed

## Purpose

Architecture Decision Records document important technical decisions and the reasoning behind them.

## ADR Format

Each ADR should include:

- Decision
- Context
- Options considered
- Consequences
- Status
- Related stories

## Example ADRs

- Why PostgreSQL is used
- Why FastAPI is used
- Why Pydantic response models are used
- Why tagged releases are validated by ted
- Why service accounts are planned
- Why API Gateway is planned
