# TERM-CS-008 Context Package

## Definition

A context package is a requirements artifact that gathers the key background, business goal, scope, architecture, stories, data models, APIs, glossary terms, and success criteria for a project area.

## Purpose

Context packages help humans and AI understand why a set of requirements exists and how the pieces fit together.
