# Role Code Registry Standard

## Purpose

The role-code registry defines the controlled vocabulary for role codes.

## Role code examples

```text
ba
sba
ba-ai
sba-ai
ba-appsupport
sba-appsupport
ba-pm
sre
```

## Rules

- Role codes should be readable and short.
- Role codes should describe role identity, not every keyword in the JD.
- Broader JD keywords should later become capabilities, not filename components.
- Role code maps to a recommended resume family.
