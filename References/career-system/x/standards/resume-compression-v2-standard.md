# Resume Compression v2 Standard

## Purpose

Reduce narrative master resumes toward two-page Teal export output while preserving the strongest evidence.

## Rules

- Core Competencies should be one compact line, not a full duplicate of Skills.
- Skills should list concrete tools, technologies, domains, and methods.
- The most recent role may use up to 8-9 bullets.
- Mid-career narrative groups should use 3-6 bullets.
- Older roles should be grouped and compressed.
- Publications should stay out of Teal-targeted resumes unless directly relevant.
- Course names should not appear as certifications unless they are actual certifications.

## Priority Evidence

Preserve:

- FRBNY enterprise financial application support
- AC Plus to OPS360 modernization
- REST API validation against Oracle
- Release readiness and deployment validation
- Runbooks and production support
- Resiliency and DR readiness
- Insurance HP PPM workflows
- Financial data platforms and golden-copy data
