I just had my interview with Mac(senior engineer/director/tech guru) and Ian (Ian's boss who is more on the business side). I introduced myself with my most recent experience at the NY Fed working on a streaming market data system that processes and distributes real-time price quotes to internal consuming applications from on-prem systems through AWS cloud infrastructure. Ian asked me more about my role in the development group and I talked about requirements and working with the developers to automate the deployment of the application in the Linux server environment through a linux ksh script that guides the devops engineer and dba to execute deployment commands in a systematic way to minimize post deployment issue. I also explained that that the application I worked on was running on 2 nodes (primary/dr) where the replication server ensures that the DR environment is in synch with the primary server at all times which is where the raw fixed income asset classes reference and price quotes data is loaded.  About performance monitoring experience, I said I have working knowledge of the Graphana dashboard while working with Operations support and the DevOps teams. (I didn't sound too confident there). Mac asked about Python and I said I use C and Java earlier in my career, i did shell scripting in bash ksh to automate manual processes whenever possible. He asked and I said, " Yes, I can read through code and pick up Python if needed" Ian asked how do you deal with a difficult customer and I said, remain calm, do structure analysis and communicate in a clear and consistent manner. Ian then pointed out that "I would add as first step - ask the team" and I felt it with good intent but I regretted not having thought of it and said it" I want to let him know that I appreciated it but yeah my bad! 

Mac asked me a `ps` question and tcp/upd. so far I did ok. at the end, I managed to answer all his questions but not all confidently. I want to reinforce my interest in the role because I always was interested in getting the experience in more upstream application closer the the exchange like inRush Ticker plant. I told everybody that the streaming market data application I worked on is more upstream where my system starts with Bloomberg feed as raw/source which we then normalize and consolidate (and cleanse) before distributing. I then said the system is similar to Redline feed where instead of using InRish Ticket plant my application used Bloomberg which must have its own ticker plant. My system at the Fed used Refinitv's TREP server as the subscribing server which is similar to pico's subscribing server to which client applications subscribe to receive updates.
